top-nav

People navigate. The system observes, connects, and improves.

top-nav is a Top Navigation Object Architecture. It uses familiar browser-like navigation patterns while the system manages objects, relationships, routing, state, usage, and learning underneath.

What This Is

This is not a navbar component. This is a reusable architecture pattern for turning visible navigation into a system-aware control layer.

User Sees

Tabs, groups, labels, pages, search, active state.

System Sees

Objects, routes, relationships, intent, state, movement, and friction.

System Learns

Usage patterns, priority, dead paths, repeated behavior, and navigation improvements.

Object: TopNav

TopNav is the visible navigation surface and the root object of this pattern. It organizes TabGroups and TabObjects into a human-readable control system.

Object: #Tab

A #Tab is not just a link. It is the smallest visible navigation object. It routes users to content while giving the system a readable signal of intent.

visible label route state relationships usage signal priority

Closed Loop

User selects tab
  ↓
System identifies intent
  ↓
Route loads content
  ↓
Relationships are mapped
  ↓
Activity is observed
  ↓
Friction is detected
  ↓
Navigation is refined

This turns static navigation into a living system.

Core Schema

type TopNav = {
  id: string;
  label: string;
  groups: TabGroup[];
  activeTabId?: string;
  createdAt: string;
  updatedAt: string;
};

type TabGroup = {
  id: string;
  label: string;
  purpose:
    | "navigation"
    | "workflow"
    | "knowledge"
    | "operations"
    | "analytics";

  tabs: TabObject[];
  order: number;
};

type TabObject = {
  id: string;
  label: string;
  slug: string;
  route: string;

  tabType:
    | "primary"
    | "secondary"
    | "utility"
    | "system";

  state:
    | "active"
    | "inactive"
    | "hidden"
    | "archived";

  visibility:
    | "public"
    | "private"
    | "role_based"
    | "state_based";

  relatedObjectIds: string[];
  priority: number;

  createdAt: string;
  updatedAt: string;
};

type SystemObservation = {
  tabId: string;

  event:
    | "viewed"
    | "clicked"
    | "ignored"
    | "searched_from"
    | "exited_from";

  frictionSignal:
    | "none"
    | "low"
    | "medium"
    | "high";

  timestamp: string;
};

Failure Modes

Static Tabs

Tabs exist only as labels. The system learns nothing.

Object Leakage

Users are forced to understand backend structure instead of navigating naturally.

No Feedback Loop

Activity is recorded but never used to improve navigation.

Deployment Rule

Every visible tab must map to a system object. Every system object must create routing, relationship, learning, or operational value.
