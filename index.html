Foundation Objects Loop Rules

Top Nav Architecture

A system pattern where people navigate through simple tabs while the platform manages objects, relationships, movement, and feedback underneath.

The visible layer stays familiar. The system layer stays structured.

Core Statement

People navigate. The system observes, connects, and improves.

Top Nav is not a navigation bar. It is a pattern for turning navigation into a system interface.

TopNav

The visible navigation surface.

TabGroup

A collection of related tabs.

TabObject

The smallest navigable system object.

ContentRoute

The destination controlled by a tab.

SystemSignal

The activity produced by navigation.

Tab Object

type TabObject = {
  id: string;
  label: string;
  slug: string;

  route: string;

  groupId?: string;

  state:
    | "active"
    | "inactive"
    | "hidden"
    | "archived";

  visibility:
    | "public"
    | "private"
    | "role_based"
    | "state_based";

  relatedObjectIds: string[];

  priority: number;

  createdAt: string;
  updatedAt: string;
};

Operating Loop

User selects a tab
  ↓
System loads the route
  ↓
Context is identified
  ↓
Relationships are mapped
  ↓
Activity is recorded
  ↓
Signals are evaluated
  ↓
Navigation is refined

System Rules

Rule 1

Every visible tab must represent a system object.

Rule 2

Every tab must control a route.

Rule 3

Every route must produce a signal.

Rule 4

Every signal must be available for improvement.

Failure Modes

Static Navigation

Tabs exist but do not produce system value.

Broken Routing

Tabs send users somewhere without preserving context.

No Signal Layer

The system records movement but does not learn from it.

Object Leakage

Users are forced to understand internal structure.

Top Nav Architecture — simple navigation, structured intelligence.
