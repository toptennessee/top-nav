Capture first. Navigate always.

top-nav is a universal workspace pattern that turns hashtags, tabs, routing, and memory into one reusable operating layer.

It is extracted from C10, but it is not limited to C10. It can be applied to any system that needs fast capture, clean navigation, object routing, and self-improving workspace structure.

Correction

This is not a navbar.

The visible tabs are only the surface. The real object is a workspace shell: a reusable interface that lets humans capture naturally while the system organizes, routes, remembers, and improves underneath.

User Sees

Hashtags, tabs, active states, clean capture, grouped workspaces.

System Sees

Objects, routes, relationships, states, intent, priority, and ownership.

System Improves

Repeated paths, friction, ignored tabs, useful shortcuts, and better defaults.

Capture Layer

Hashtags are commands.

A hashtag is not decoration. It is a lightweight routing instruction. It tells the workspace how to classify, store, and surface the captured object. 

#idea     → create idea object
#task     → create action object
#note     → create knowledge object
#project  → create project object
#system   → create system object
#archive  → close or store object
Navigation Layer

Tabs are live workspaces.

A tab is not only a place to click. It is a workspace view with its own route, state, memory, and object relationships. 

TabWorkspace
  label: "#projects"
  route: "/projects"
  accepts: ["#task", "#note", "#system"]
  state: "active"
  memory: "recent project objects"
  improvement: "promote repeated paths"
System Loop

Show and tell in one interface.

The interface shows simple navigation while telling the system what the user is trying to do. Every capture and every tab click becomes a signal.

Memory Layer

The workspace remembers movement.

A normal template displays sections. top-nav watches how work moves between sections, then uses that signal to improve the next route. 

WorkspaceMemory = {
  recentCaptures: [],
  repeatedRoutes: [],
  ignoredTabs: [],
  highFrictionPaths: [],
  promotedTabs: [],
  archivedObjects: []
}
Object Model

Universal core schema.

type WorkspaceShell = {
  id: string;
  name: string;
  captureBar: CaptureBar;
  tabs: TabWorkspace[];
  memory: WorkspaceMemory;
  createdAt: string;
  updatedAt: string;
};

type CaptureBar = {
  id: string;
  placeholder: string;
  acceptedCommands: HashtagCommand[];
  defaultRoute: string;
};

type HashtagCommand = {
  tag: string;
  objectType:
    | "idea"
    | "task"
    | "note"
    | "project"
    | "system"
    | "archive";

  routeTo: string;
  createsObject: boolean;
};

type TabWorkspace = {
  id: string;
  label: string;
  route: string;
  acceptedTags: string[];
  state:
    | "active"
    | "inactive"
    | "pinned"
    | "hidden"
    | "archived";

  relatedObjectIds: string[];
  priority: number;
};

type RoutedObject = {
  id: string;
  sourceText: string;
  detectedTags: string[];
  objectType: string;
  currentRoute: string;
  relatedObjectIds: string[];
  status:
    | "captured"
    | "routed"
    | "active"
    | "resolved"
    | "archived";

  createdAt: string;
};

type WorkspaceSignal = {
  event:
    | "capture_created"
    | "tab_opened"
    | "object_moved"
    | "search_used"
    | "route_repeated"
    | "tab_ignored";

  objectId?: string;
  tabId?: string;
  friction:
    | "none"
    | "low"
    | "medium"
    | "high";

  timestamp: string;
};
Deployment Rule

Every visible action must produce system value.

If it can be captured, it should be routed. If it can be routed, it should be remembered. If it can be remembered, it should improve the workspace.

Routing Value

Where should this object go?

Relationship Value

What is this connected to?

Learning Value

What should the workspace do better next time?
