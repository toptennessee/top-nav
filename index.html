top-nav
Universal capture-navigation workspace

Capture first. Navigate always.

top-nav is a universal workspace pattern that turns capture, primary layers, #Tab Objects, routing, and memory into one reusable operating layer.

It is extracted from C10, but it is not limited to C10. It can be applied to any system that needs fast capture, clean routing, object destinations, and self-improving workspace structure.

Correction

This is not a navbar.

The visible interface is a capture shell. It behaves like a mini browser: primary layers sit across the top, and each layer exposes #Tab Objects that route captured work.

A capture button, clean popup, primary layers, and selectable #Tab Objects.

System Sees

Capture intent, object type, route, relationship, priority, and state.

System Improves

Repeated routes, useful destinations, ignored layers, and better defaults.

Primary Layers

Layers organize the capture space.

Primary layers behave like a familiar bookmarks-bar pattern, but they are not links. They are capture categories that expose valid object destinations. 

PrimaryLayer = {
  label: "Systems",
  purpose: "organize system-level captures",
  exposes: ["#model", "#loop", "#spec", "#pattern"]
}
#Tab Objects

#Tabs are routing destinations.

A #Tab Object is not a menu item. It tells the system where the captured object belongs and what kind of operational value it should create. 

#idea     → route to idea object
#task     → route to action object
#model    → route to system model
#loop     → route to feedback loop
#archive  → route to stored memory

Core Schema

Universal workspace object model.

type CaptureShell = {
  id: string;
  name: string;
  captureInput: string;
  primaryLayers: PrimaryLayer[];
  activeLayerId?: string;
  selectedTabObjectId?: string;
  memory: WorkspaceMemory;
};

type PrimaryLayer = {
  id: string;
  label: string;
  purpose: string;
  tabObjects: TabObject[];
  order: number;
};

type TabObject = {
  id: string;
  label: string;
  hashtag: string;
  route: string;

  objectType:
    | "idea"
    | "task"
    | "note"
    | "project"
    | "system"
    | "archive";

  creates:
    | "routing_value"
    | "relationship_value"
    | "learning_value"
    | "operational_value";

  state:
    | "available"
    | "selected"
    | "hidden"
    | "archived";
};

type RoutedCapture = {
  id: string;
  sourceText: string;
  primaryLayerId: string;
  tabObjectId: string;
  route: string;
  status:
    | "draft"
    | "submitted"
    | "routed"
    | "remembered";
};

type WorkspaceMemory = {
  recentRoutes: string[];
  repeatedDestinations: string[];
  ignoredLayers: string[];
  promotedTabObjects: string[];
};

Every capture must land somewhere useful.

Primary layers organize the capture space. #Tab Objects route the captured object. Submit commits the object into memory.
