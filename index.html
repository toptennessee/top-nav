Overview #Tab Object Top Nav System Loop Schemas

C10 Tab Object System

People navigate. The system observes, connects, and improves.

C10 is a bridge between human navigation and system intelligence. Users interact with familiar top-nav tabs. Underneath, each tab is treated as a system object with relationships, state, routing behavior, analytics, and optimization signals.

Core Distinction

User Sees

Tabs, sections, categories, pages, activity, navigation.

System Sees

Objects, relationships, retrieval paths, permissions, states, usage patterns, and operational intent.

Object: #Tab

Purpose

A #Tab is the smallest navigable unit in C10. It looks like a simple navigation label to the user, but functions as an addressable object inside the system.

Correction

A tab is not just a link. A tab is a controlled interface object that routes humans toward content while routing the system toward context.

Object: TopNav

Purpose

TopNav organizes active #Tab objects into a visible human navigation layer. It is the user-facing map of the workspace. 

TopNav
  ↓
TabGroup
  ↓
#Tab
  ↓
ContentRoute
  ↓
SystemObservation

Closed Loop

Human clicks tab
    ↓
System identifies intent
    ↓
Content route is loaded
    ↓
Relationships are mapped
    ↓
Usage signal is recorded
    ↓
Navigation is refined
    ↓
TopNav improves

This turns static navigation into a live control system.

Object Schemas

TabObject

type TabObject = {
  id: string;

  label: string;

  slug: string;

  tabType:
    | "primary"
    | "secondary"
    | "utility"
    | "system";

  visibility:
    | "public"
    | "private"
    | "role_based"
    | "state_based";

  state:
    | "active"
    | "inactive"
    | "hidden"
    | "archived";

  route: string;

  parentGroupId?: string;

  relatedObjectIds: string[];

  priority: number;

  createdAt: string;
  updatedAt: string;
};

TabGroup

type TabGroup = {
  id: string;

  label: string;

  purpose:
    | "navigation"
    | "workflow"
    | "knowledge"
    | "operations"
    | "analytics";

  tabs: TabObject[];

  displayMode:
    | "top_nav"
    | "sidebar"
    | "dropdown"
    | "contextual";

  order: number;
};

SystemObservation

type SystemObservation = {
  tabId: string;

  event:
    | "viewed"
    | "clicked"
    | "ignored"
    | "searched_from"
    | "exited_from";

  userIntent?: string;

  frictionSignal:
    | "none"
    | "low"
    | "medium"
    | "high";

  timestamp: string;
};

Failure Modes

Static Tabs

Navigation becomes decorative and does not teach the system anything.

Object Leakage

Users are forced to understand backend concepts instead of navigating naturally.

No Feedback Layer

The system records activity but never improves routing, grouping, or priority.

Deployment Rule

Every visible tab must have a system object behind it. Every system object must produce routing, relationship, or learning value.
